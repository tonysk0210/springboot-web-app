課程資料查詢與新增表單綁定

在課程管理頁面中，AdminController 透過 @ModelAttribute("courseList") 方法 displayAllCourses() 將目前所有已存在的 List<Course> 資料傳遞至前端進行渲染。 

@ModelAttribute("courseList")
public List<Courses> displayAllCourses() {
    // 從資料庫中查詢所有 Course 物件，並根據 courseId 欄位以「升冪排序（ascending）」的方式回傳結果。
    return coursesRepository.findAll(Sort.by(Sort.Direction.ASC, "courseId"));
}

此外，系統也會透過 Model 提供一個新的 Course 物件， 供「新增課程」表單進行資料綁定使用，作為新增課程時的資料輸入對象。

課程新增流程

管理員可透過表單輸入課程名稱與費用，這些欄位會綁定至 Course 物件。 表單送出後，Controller 會執行後端驗證，若資料有效，則將該課程存入資料庫中的 course 資料表。coursesRepository.save(course); 驗證條件如下：
name：課程名稱必填
fees：課程費用必填｜僅允許純數字

課程新增成功後，系統將重新導向至課程管理頁面，並重新渲染更新後的課程清單。 每個課程項目均會附帶「查看」與「刪除」按鈕，其連結中包含 courseId 作為 URL 查詢參數（如：?courseId=123）。

刪除課程流程：解除學生關聯後移除課程資料

當管理員點選「刪除」後，系統會根據 URL 中的 courseId，透過 @RequestParam 取得對應的 Course 物件。 隨後，系統會遍歷該課程所關聯的 Set<Person>（即所有已選修此課程的學生 ，並逐一解除每位學生與該課程之間的關聯 person.getCourses().remove(course) 。 完成關聯解除後，系統會呼叫 courseRepository.delete(course)，將該課程從資料庫移除。 

// @ManyToMany 表示 Person 可選修多門課程，Course 也可被多位 Person 選修
@ManyToMany(mappedBy = "courses", fetch = FetchType.EAGER) // mappedBy 表示關聯的擁有端在 Person 類別的 courses 屬性
private Set<Person> persons = new HashSet<>();

// 擁有端負責維護關聯關係並主導中介表（JoinTable）的結構與更新操作
Optional<Courses>  courseToBeDeletedOptional = coursesRepository.findById(courseId);
Courses courseToBeDeleted = courseToBeDeletedOptional.get(); // 取得要刪除的 Course 物件 
for (Person person : courseToBeDeleted.getPersons()) {       // 逐一處理所有註冊此課程的學生
    person.getCourses().remove(courseToBeDeleted);           // 從每位學生的 Courses 集合中移除該課程（解除關聯） 
    personRepository.save(person);                           // 儲存變更，更新資料庫中的關聯 (可省略)

coursesRepository.deleteById(courseId);                      // 全部解除關聯後，刪除該課程
