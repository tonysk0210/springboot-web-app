Spring Security 登入後導向 Dashboard 並載入使用者資料

登入成功後，使用者將依其角色（Admin 或 Student）導向個人 Dashboard，該頁面由 DashboardController 負責處理。

系統會透過 Spring Security 所提供的 Authentication 物件，取得目前登入者所對應的 Person 實體，並進一步擷取使用者姓名、角色（Role）、以及所屬方案（Plan）等資訊，將其渲染於 Dashboard 畫面中。 

Person person = personRepository.readByEmail(authentication.getName()); // 根據登入的 email 從資料庫取得 Person 物件

model.addAttribute("username", person.getName()); // 將使用者名稱傳送至前端作為 username
model.addAttribute("role", authentication.getAuthorities().toString()); // 傳送使用者角色集合至前端作為 role
if (person.getPlan() != null)
    model.addAttribute("plan", person.getPlan().getName()); // 若使用者有加入方案，傳送方案名稱至前端作為 plan

Thymeleaf 與 Spring Security：依角色/登入狀態動態顯示內容

為實現依角色呈現功能模組，前端畫面透過 Thymeleaf 的 Spring Security 擴充屬性 sec:authorize 進行條件判斷，例如： 

<div sec:authorize="hasRole('ROLE_ADMIN')">
    <!-- 只有管理者角色可見的區塊 -->
</div>

<div sec:authorize="hasRole('ROLE_STUDENT')">
    <!-- 只有學生角色可見的區塊 -->
</div>

NavBar 也可根據登入狀態動態顯示Login或Logout選項： 

<a sec:authorize="isAnonymous()" th:href="@{/login}">Login</a> <!-- 僅限匿名訪客可見 -->

<a sec:authorize="isAuthenticated()" th:href="@{/logout}">Logout</a> <!-- 僅限已登入使用者（包含管理員與學生）可見 -->

登入者資訊保存至 Session

為提升資料重用性與操作效率，系統會將登入者的 Person 物件儲存至 HttpSession ("loggedInPerson") 中，讓後續其他 Controller 可直接從 Session 中取得使用者資訊，避免重複查詢資料庫。 

session.setAttribute("loggedInPerson", person); // 將登入成功的使用者資訊存入 Session 中

實現非表單（非 POST）的登出邏輯

Logout — 由於 Spring Security 的內建登出機制必須透過 POST 方法（表單方式）發送請求，若希望以 連結（超連結）方式 實現登出功能，可改為透過 @Controller 手動處理。

透過建立 SecurityContextLogoutHandler，並傳入當前使用者的 Authentication 物件，即可完成登出流程。 完成後，系統會 redirect 至 /login?logout=true，由前端判斷 URL 參數，顯示 You have logged out successfully 訊息。 

@GetMapping("/logout")
public String logout(HttpServletRequest request, HttpServletResponse response, Authentication authentication) {
     // 1) 建立 SecurityContextLogoutHandler 實例
    SecurityContextLogoutHandler logoutHandler = new SecurityContextLogoutHandler();

     // 2) 呼叫 logout() 方法以：
     // - 清除 SecurityContext 中的 Authentication 物件
     // - 使 HTTP Session 失效
     // - 刪除名為 JSESSIONID 的 Cookie
    logoutHandler.logout(request, response, authentication);

    return "redirect:/login?logout=true";
}
