學生選課頁面渲染與已註冊課程判斷邏輯

@GetMapping("/signUpCourses")
public String signUpCourses(Model model, HttpSession session) {
    List<Courses> courseList = coursesRepository.findAll();  // 取得所有已開設的課程 List<Courses>
    model.addAttribute("courseList", courseList);  // 將 courseList 加入 Model，供前端顯示課程清單供學生選購

    Person person = (Person) session.getAttribute("loggedInPerson");  // 從 Session 中取得當前登入的學生 Person 物件 
    model.addAttribute("alreadyRegisteredCourses", person.getCourses());  // 將學生已註冊的課程集合（Set<Courses>）加入 Model，供前端與開設課程對照使用
    return "/authenticated/studentOnly/signUpCourses";  // 導向學生專屬的選課頁面
}

此時前端頁面會以迴圈方式渲染所有課程。若某課程已存在於學生的 Set<Course> 中，則代表該課程已被此學生註冊， 對應的勾選框（checkbox）將自動設為 disabled 狀態，以防止學生重複選課。 每個 checkbox 的 value 對應該課程的 courseId。 

<tr th:each="course : ${courseList}"> <!-- 對所有課程進行迴圈，依序顯示每門課的 courseId、name 與 fees -->
    <td th:text="${course.courseId}"></td>
    <td th:text="${course.name}"></td>
    <td th:text="${course.fees}"></td>
    <td>
        <input type="checkbox" name="selectedCourses"
           th:value="${course.courseId}" <!-- 每個勾選框代表一個 courseId -->
           th:disabled="${#lists.contains(alreadyRegisteredCourses, course)}"/> <!-- 若學生已註冊此課程，則禁用該勾選框（避免重複選課）-->
    </td>
</tr>

學生選課提交處理流程與成功／錯誤訊息呈現邏輯

@PostMapping("/purchaseSelectedCourse")
public String purchaseSelectedCourses(@RequestParam(value = "selectedCourses", required = false) List<Integer> listOfCourseIds, HttpSession session, RedirectAttributes redirectAttributes) {
    if (listOfCourseIds == null || listOfCourseIds.isEmpty()) { // 若學生沒有勾選任何課程(listOfCourseIds 為 courseId集合)
        redirectAttributes.addFlashAttribute("error", "Please select at least one course"); // 顯示錯誤訊息「請至少勾選一門課程」
        return "redirect:/student/signUpCourses";
    }

    Person person = (Person) session.getAttribute("loggedInPerson");　// 從 Session 中取得當前登入的學生 Person 物件

    for (Integer courseId : listOfCourseIds) {
        person.getCourses().add(coursesRepository.findById(courseId).get()); // 根據所選的 courseId，將課程加入 Person 的課程清單（由關聯的擁有端進行維護）
    }

    Person updatedPerson = personRepository.save(person); // 儲存更新後的 Person（即課程關聯變更）至資料庫
    session.setAttribute("loggedInPerson", updatedPerson); // 更新 Session 中的 loggedInPerson 為最新狀態
    return "redirect:/student/signUpCourses?success"; // 重新導向回選課頁面，並透過 URL 參數通知前端顯示「選課成功」訊息
}

若該列表為 null 或為空（代表未選課），系統會 redirect 回同一頁， 並顯示提示訊息: ⚠️ Please select at least one course

更新完成後，系統會重新導向回原頁面，顯示成功選課信息: ✅ Your purchase was successful 並渲染最新的課程清單與學生已註冊課程狀態。
