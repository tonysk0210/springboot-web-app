使用者聯絡表單提交流程與驗證機制（Spring Boot + JPA + Thymeleaf 整合）

/contact 路徑由 ContactController 負責接收並處理請求。 系統會先建立一個新的 Contact 物件，並透過 Model 與前端表單欄位進行屬性綁定（MVC Binding)。 使用者提交表單後， Contact 物件經由 /saveMessage 路徑回傳至 ContactController 做後續驗證。

Spring Boot Validation 套件根據 Contact 類別中針對所定義的驗證條件進行驗證。 驗證條件如下：若驗證未通過，將於前端顯示相對應的錯誤信息並要求重新輸入；

  • name：不得為空白 | ⚠️Name must not be blank⚠️
  • mobile：必須為 10 碼數字 | ⚠️Mobile Number must be exactly 10 digits long⚠️
  • subject：不得為空白 | ⚠️Subject must not be blank⚠️
  • message：不得為空白 | ⚠️Message must not be blank⚠️

若通過驗證，Contact 物件將交由 ContactService 設定其 status 屬性為 OPEN（代表尚未讀取）， 接著透過 ContactRepository，將資料儲存至 contact_msg 資料表中 ( Spring Data JPA ORM機制 )。完成儲存後，系統會使用 RedirectAttributes 設定 Flash Attribute：redirectAttributes.addFlashAttribute("successfullySavedMessage", "Message successfully submitted"); 並重執行 redirect:/contact， 由Thymeleaf 顯示成功提交信息 | Message successfully submitted

審計機制說明：Contact 實體的自動記錄與追蹤機制

Contact 類別繼承自 BaseEntity，作為啟用審計功能的實體類別，當 Spring Data JPA 執行 repository.save() 時，系統將自動產生 createdAt、 createdBy、updatedAt、updatedBy 等欄位，並存入資料表中。其中，時間相關欄位由資料庫的內建時間機制自動填入；而建立者與更新者資訊則由 AuditorAware 的實作類別決定：此實作會以 Authentication 物件為依據。 若使用者已登入，將記錄 authentication.getName() (這裡為使用者的 email)，否則預設為 "anonymousUser"。 

@Component("auditAwareImpl")
public class AuditAwareImpl implements AuditorAware<String> { // 需搭配 Spring JPA 使用

    @Override
    public Optional<String> getCurrentAuditor() {
        // 取得當前使用者的 Authentication 物件
        Authentication authentication = SecurityContextHolder.getContext().getAuthentication();

        if (authentication == null || !authentication.isAuthenticated()) {
            // 若尚未登入，回傳 "anonymousUser"：createdBy / updatedBy 欄位將記錄此值
            return Optional.of("anonymousUser");
        }

        // 若已登入，回傳使用者名稱（通常是 email），記錄於 createdBy / updatedBy
        return Optional.of(authentication.getName());
    }
}
