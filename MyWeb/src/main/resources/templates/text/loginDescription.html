自訂登入流程：整合 Spring Security 與表單驗證的登入設定

Login 功能 <form th:action="@{/login}" method="post"> 表單送出後，將由 Spring Security 自動攔截請求，並進一步透過其內建的 filterChain(HttpSecurity http, AuthenticationProvider myCustomProvider) 方法進行處理，該方法會回傳一個 SecurityFilterChain Bean ，在此方法中，可透過 HttpSecurity.formLogin() 配置整體登入流程，包含登入成功與失敗後的導向路徑設定，實現自訂化的登入體驗。

@Bean
SecurityFilterChain filterChain(HttpSecurity http, AuthenticationProvider myCustomProvider)

http.formLogin(loginConfig -> loginConfig
                .loginPage("/login") // Spring Security 將自動攔截 POST 請求至 /login
                .defaultSuccessUrl("/dashboard") // 登入成功後導向 /dashboard
                .failureUrl("/login?error=true")); // 登入失敗後導向 /login，並附加錯誤參數

http.authenticationProvider(myCustomProvider); // 註冊自定義的 AuthenticationProvider 負責驗證邏輯

自訂驗證邏輯：實作 AuthenticationProvider 建立使用者驗證流程

使用者提交的帳號與密碼會由自訂的 AuthenticationProvider（註冊為 @Component）負責驗證。該元件內的 authenticate(Authentication authentication) 方法會比對使用者名稱與經由 PasswordEncoder（如 BCrypt）雜湊處理的密碼是否一致。

@Component
public class UsernamePwdAuthenticationProvider implements AuthenticationProvider

@Override
public Authentication authenticate(Authentication authentication) {

    String enteredEmail = authentication.getName(); // 表單所輸入的 email
    String enteredPassword = authentication.getCredentials().toString(); // 表單所輸入的密碼

    Person person = personRepository.readByEmail(enteredEmail); // 透過輸入的 email 從資料庫取得對應的 Person 物件
    if (person != null && passwordEncoder.matches(enteredPassword, person.getPassword()))  // 驗證 Person 是否存在，且輸入密碼與加密後的密碼是否相符
        return new UsernamePasswordAuthenticationToken(
                enteredEmail,
                null,
                getGrantedAuthorities(person.getRoles())); // 驗證成功，建立 Authentication 物件並附帶使用者角色權限
    else
        throw new BadCredentialsException("Invalid credentials"); // 驗證失敗，拋出例外，由 Spring Security 處理導向登入失敗頁面
}

若驗證成功，系統會為該使用者（Person）建立一個 Authentication 物件，實作類型為 UsernamePasswordAuthenticationToken， 其中包含以下資訊：

  • principal：代表使用者本身（此處為 Email）
  • credentials：使用者的密碼（驗證用，登入後不再儲存）
  • authorities：使用者擁有的角色或權限（ROLE_STUDENT 或 ROLE_ADMIN）

此 Authentication 物件隨後會被儲存至 SecurityContextHolder，代表使用者已成功登入並建立安全性上下文。登入成功後，系統將自動導向 /dashboard。 若無法取得對應的使用者物件，或密碼驗證失敗，則登入流程會導向 /login?error=true。前端登入頁面將透過 Thymeleaf 根據 URL 參數 error=true 顯示錯誤訊息：
Invalid Email or Password
