View Plan Detail：顯示方案資訊與所屬學生清單

當管理員點選 「View」 按鈕後，系統將導向 /admin/viewPlanDetail?planId={?} 路徑，AdminController 會透過 @RequestParam 接收對應的 planId，並呼叫 planRepository.findById(planId) 查詢該方案的 Plan 物件。 查詢結果將透過 Model 傳送至前端，用於動態渲染該方案名稱及其所屬學生清單。

學生清單為 Set<Person> 型別，透過 Thymeleaf 的迴圈語法渲染至畫面： 

<tr th:each="person : ${thePlan.persons}"> <!-- 迴圈渲染 Set<Person> 中的每位學生 -->
    <td th:text="${person.name}"></td>
    <td th:text="${person.email}"></td>
    <td th:text="${person.mobile}"></td>
    <td class="actions">
        <a th:href="@{/admin/removeStudent(personId=${person.personId})}"
           onclick="return confirm('Are you sure you want to remove this student?')">Remove</a> <!-- Remove按鈕 點擊後確認移除該學生 -->
    </td>
</tr>

此區塊將列出所有隸屬於該方案的學生資訊。

此外，系統也會建立一個新的 Person 物件，透過 Model 傳送至前端，作為 「Add Student」 表單的資料綁定對象。 為了能將新增的學生正確加入該方案，系統會將目前查詢到的 Plan 物件儲存至 HttpSession ("thePlanInSession") 中，方便在後續處理新增學生的流程中使用。

從方案中添加學生：建立 Person 與 Plan 的關聯操作

管理員可透過輸入學生的 Email，將該學生加入至目前方案中：

系統會比對輸入的 Email 是否存在於 Person 資料表中： 

Person thePerson = personRepository.readByEmail(person.getEmail()); // 若找不到符合的 email，則回傳 null

若查無對應學生，系統將重新導向回原頁面，並附加錯誤參數 ?error=personNotFound ，由 Thymeleaf 顯示錯誤訊息：
⚠️ Student not found. Please enter a valid email.

若成功找到對應學生，系統會先呼叫： 

Plan thePlanInSession = (Plan) session.getAttribute("thePlanInSession"); // 取得目前參照的方案（inverse side）
thePerson.setPlan(thePlanInSession); // 由擁有端（Person）設定與方案的關聯關係

將學生與方案建立關聯後，接著透過 

personRepository.save(thePerson); // 由擁有端（Person）執行關聯更新

更新資料庫，此動作會同步更新 Person 的 plan_id 外鍵。 完成操作後，系統將重新導向至原頁面，並重新取得該方案的學生清單以進行渲染。

從方案中移除學生：解除 Person 與 Plan 的關聯操作

若管理員欲移除學生，系統將執行以下操作流程：

根據 「Remove」 按鈕所附帶的 URL 參數 personId，查詢對應的 Person 物件。 

Optional<Person> thePersonToBeRemovedOptional = personRepository.findById(personId);
Person thePersonToBeRemoved = thePersonToBeRemovedOptional.get(); // 取得要移除的 Person 物件
Plan thisPlanToBeRemovedFrom = thePersonToBeRemoved.getPlan(); // 取得目前所屬的 Plan 物件（Person 擁有 plan_id 作為外鍵，因此僅能隸屬於一個 Plan）
thePersonToBeRemoved.setPlan(null); // 由擁有端解除關聯
personRepository.save(thePersonToBeRemoved); // 儲存解除關聯後的變更

重新導向當前頁面 "redirect:/admin/viewPlanDetail?planId=" + thisPlanToBeRemovedFrom.getPlanId(); 重新渲染所屬學生清單。
